## 4. 用户流程

### 4.1 完整流程图

```
用户访问网站 --> 已登录?
    │
    ├── 否 --> 显示登录页面 --> 用户点击Google登录 --> 跳转Google授权 --> 授权成功返回
    │
    └── 是 --> 检查剩余次数
                  │
                  ├── < 1次 --> 提示:次数用完 --> 引导付费/明日再来
                  │
                  └── >= 1次 --> 用户上传图片 --> 图片验证
                                              │
                                              ├── 失败 --> 显示错误提示
                                              │
                                              └── 成功 --> AI处理图片
                                                            │
                                                            ├── 失败 --> 显示错误+重试
                                                            │
                                                            └── 成功 --> 展示结果 --> 扣减使用次数 --> 用户下载图片
```

### 4.2 异常处理清单

| 场景 | 用户提示 | 处理方式 |
|------|----------|----------|
| 未登录 | "请先登录后使用" | 跳转登录页 |
| 次数用完 | "今日免费次数已用完" | 引导付费/明日再来 |
| 文件太大 | "图片不能超过 10MB" | 提示缩小图片 |
| 格式不支持 | "仅支持 JPG/PNG/WebP" | 提示支持的格式 |
| AI处理失败 | "处理失败，请重试" | 自动重试1次，仍失败提示联系客服 |
| 网络超时 | "网络不稳定，请重试" | 自动重试 |
| 登录失败 | "登录失败，请重试" | 显示 Google 登录按钮 |

## 5. 技术方案

### 5.1 技术架构

```
┌─────────────────────────────────────────────────────────┐
│                     前端 (Cloudflare Pages)             │
│   ┌─────────┐  ┌─────────┐  ┌─────────────────────┐   │
│   │ 静态页面 │  │ JS交互  │  │  Google登录SDK      │   │
│   └────┬────┘  └────┬────┘  └──────────┬──────────┘   │
└────────┼────────────┼──────────────────┼───────────────┘
         │            │                   │
         │            ▼                   │
         │      ┌─────────────────────┐   │
         │      │  后端 (Cloudflare   │   │
         │      │     Workers)         │   │
         │      ├─────────────────────┤   │
         │      │  登录接口           │   │
         │      │  图片处理接口       │   │
         │      │  限流检查           │   │
         │      └──────────┬──────────┘   │
         │                 │              │
         ▼                 ▼              ▼
┌─────────────────────────────────────────────────────────┐
│              第三方服务                                  │
│   ┌─────────────┐     ┌─────────────────────────┐    │
│   │ Google OAuth │     │  Workers AI (BiRefNet)  │    │
│   └─────────────┘     └─────────────────────────┘    │
│                                                         │
│   ┌─────────────────────────────────────────┐          │
│   │       Cloudflare KV (用户限流)          │          │
│   └─────────────────────────────────────────┘          │
└─────────────────────────────────────────────────────────┘
```

### 5.2 技术选型

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Cloudflare Pages + 原生 HTML/JS | 静态页面，无需后端服务器 |
| 后端 | Cloudflare Workers | 边缘函数，处理图片请求 |
| AI引擎 | Cloudflare Workers AI (BiRefNet) | 背景移除模型 |
| 用户认证 | Google OAuth 2.0 | 用户登录 + 身份识别 |
| 数据存储 | Cloudflare KV | 存储用户使用次数 |
| 部署 | Cloudflare 全球 CDN | 免费/低成本 |

### 5.3 限流策略

#### KV 存储结构

```json
{
  "user_id_google": {
    "used_today": 2,
    "last_date": "2026-03-15"
  }
}
```

#### 每日重置逻辑

每次查询时检查 `last_date`，如果是新日期则重置 `used_today = 0`

### 5.4 API 接口设计

| 接口 | 方法 | 描述 | 参数 |
|------|------|------|------|
| /auth/login | GET | Google OAuth 登录跳转 | 无 |
| /auth/callback | GET | Google OAuth 回调 | code |
| /api/usage | GET | 获取剩余次数 | 无（从session获取user_id） |
| /api/remove-bg | POST | 上传图片并处理 | image (文件) |

## 7. 成本估算

### 7.1 Cloudflare 费用明细

| 项目 | 免费额度 | 超出费用 |
|------|----------|----------|
| Workers AI | 10,000 Neurons/天 | $0.011 / 1000 Neurons |
| Workers 请求 | 10万次/天 | $0.013 / 10万次 |
| Cloudflare KV | 1GB 存储 / 10万次读写 | 免费 |
| Cloudflare Pages | 免费 | - |

### 7.2 成本预估

| 用户规模 | 每日处理量 | 预估成本/天 |
|----------|-----------|-------------|
| 100 用户 | 300 次 | $0.15-0.3 |
| 1,000 用户 | 3,000 次 | $1.5-3 |
| 10,000 用户 | 30,000 次 | $15-30 |

> 预估：1000 用户/天 × 3 次 = 3000 次/天，成本约 $1.5-3/天（约 ¥10-20/天）

## 9. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| Google OAuth 审核 | 可能需要 1-2 周 | 提前申请，使用自有账号测试 |
| Cloudflare AI 额度不够 | 高峰期服务中断 | 监控用量，超限前预警 |
| 用户刷次数 | 成本损失 | 绑定 Google 账号 + KV 记录，可后期加验证码 |
| 图片处理慢 | 用户体验差 | 优化 Workers 冷启动，增加加载动画 |
